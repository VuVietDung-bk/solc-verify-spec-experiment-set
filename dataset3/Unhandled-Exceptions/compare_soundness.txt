==============================================================================
SOUNDNESS COMPARISON — dataset3/Unhandled-Exceptions
==============================================================================
Contracts: buggy_1 (HotDollarsToken), buggy_5 (Ownable + TokenERC20 + TTC),
  buggy_19 (ethBank + owned), buggy_20 (RampInstantEscrowsPoolInterface +
  RampInstantPool), buggy_23 (AGR + ERC20 + ERC20Burnable),
  buggy_33 (Staking), buggy_34 (Ownable + LollypopToken),
  buggy_45 (StockBet), buggy_47 (AcunarIEO + AcunarToken),
  buggy_50 (digitalNotary)
Vulnerability: Unhandled exceptions — injected functions use send()/call()
  without checking the return value, have empty error handlers, inverted
  send-check logic, or always-revert conditions.

Spec strategy: Specs test the INJECTED unchecked-send/call functions
  directly using four main patterns:
  (A) Comment-only / implicit completion: Rule calls function, no assertion.
      Passes if function can complete without reverting. Documents that
      unchecked send/call allows silent failure.
  (B) assert true: Same as (A) but explicit. "Function completes."
  (C) assert payedOut: Tests state corruption — sendToWinner sets payedOut
      flag to true BEFORE calling send(), so flag is true regardless of
      send result. Property IS true.
  (D) assert_revert: Tests that function always reverts. Used for
      always-revert (|| 1==1) and inverted-logic patterns.
