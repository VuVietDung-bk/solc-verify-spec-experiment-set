==============================================================================
SOUNDNESS COMPARISON — dataset3/Overflow-Underflow
==============================================================================
Contracts: buggy_1 (HotDollarsToken), buggy_5 (Ownable + TokenERC20 + TTC),
  buggy_19 (owned + ethBank), buggy_20 (Ownable + Stoppable + RampInstantPool),
  buggy_23 (ERC20 + ERC20Burnable + AGR), buggy_33 (Owned + Staking),
  buggy_34 (Ownable + LollypopToken), buggy_45 (StockBet),
  buggy_47 (ERC20Interface + AcunarToken + AcunarIEO),
  buggy_50 (digitalNotary)
Vulnerability: Integer overflow/underflow in injected helper functions

Bug patterns:
  Type A — Transfer underflow: require(balances[sender] - value >= 0); uint
    subtraction wraps around (Solidity <0.8 semantics), so this require ALWAYS
    passes even when balance is 0. Spec requires revert (assert_revert).
  Type B — LockTime overflow: lockTime[sender] += delta; when delta is large
    enough, the addition wraps around and lockTime DECREASES. Spec asserts
    afterTime > before (monotonic increase).

All rules test BUGGY functions. All properties ARE violated by the bugs.
Every [ERROR] from either tool = TP.

--- buggy_1 / EIP20Interface (2 SVS / 2 CVL) ---
Contract: HotDollarsToken with injected transfer_intou14 (Type A underflow)
and increaseLockTime_intou13 (Type B overflow).

  transfer_intou14_should_not_underflow [ERROR/ERROR]
    balances_intou14[sender] - value wraps to large uint when balance=0,
    value>0. require(... >= 0) always passes. Function does NOT revert.
    Property IS violated (should revert but doesn't).
    SVS: TP | CVL: TP

  increaseLockTime_intou13_should_not_overflow [ERROR/ERROR]
    lockTime_intou13[sender] += delta overflows when delta is near MAX_UINT.
    afterTime wraps to value < before. Property IS violated.
    SVS: TP | CVL: TP
